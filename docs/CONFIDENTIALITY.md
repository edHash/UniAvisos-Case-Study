# Confidencialidad y alcance de la publicación

> Documentar un sistema no significa exponerlo.
> La transparencia profesional también necesita límites.

Este repositorio funciona como un **caso de estudio público** sobre la evolución técnica de UniAvisos.

UniAvisos es un proyecto oficial sujeto a lineamientos institucionales. Por esta razón, la documentación publicada se limita a describir objetivos, decisiones arquitectónicas, desafíos, pruebas y aprendizajes en un nivel conceptual.

Este repositorio no constituye una distribución del producto ni sustituye su documentación interna.

```text
[ PUBLICATION SCOPE ]

Repository type: Public case study
Source code: Restricted
Institutional data: Excluded
Real screenshots: Pending authorization
Internal infrastructure: Not disclosed
```

* Notificaciones
* Persistencia local
* Manejo de estados
* Escalabilidad

* Cómo evolucionó el sistema
* Qué resultados se validaron

# 12. Política de actualización

La documentación podrá actualizarse conforme evolucione el proyecto, siempre que:

* La información siga siendo pública
* No se revelen detalles restringidos
* Se mantenga la precisión técnica
* Se diferencie entre funciones actuales y funciones planificadas
* No se presenten planes como resultados terminados
* Las nuevas evidencias visuales estén autorizadas

Los cambios importantes deberán registrarse mediante commits descriptivos.
